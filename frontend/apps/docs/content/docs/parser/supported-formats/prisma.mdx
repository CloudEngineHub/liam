## Prisma and `schema.prisma`

When a Prisma project manages migrations using the Prisma CLI, the latest database structure is typically described in `schema.prisma`.  

When using Liam CLI, specify `--format prisma` and `--input path/to/schema.prisma` as follows:

<Tabs items={["npm", "pnpm", "yarn", "bun"]}>
  <Tab value="npm" copyable>npx @liam-hq/cli erd build --format prisma --input prisma/schema.prisma</Tab>
  <Tab value="pnpm" copyable>pnpm dlx @liam-hq/cli erd build --format prisma --input prisma/schema.prisma</Tab>
  <Tab value="yarn" copyable>yarn dlx @liam-hq/cli erd build --format prisma --input prisma/schema.prisma</Tab>
  <Tab value="bun" copyable>bun x @liam-hq/cli erd build --format prisma --input prisma/schema.prisma</Tab>
</Tabs>

If the above command runs without issue, you should see an ER diagram generated.

## Under the Hood

Liam CLI analyzes the content of your `schema.prisma` file using a dedicated parser that relies on [`@prisma/internals`](https://www.npmjs.com/package/@prisma/internals) to build the ER diagram.
